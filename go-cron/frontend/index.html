How to Test the API

  1. List Jobs: The jobs table below will automatically load all jobs. Click Refresh Jobs to reload the list at any time.
  2. Create a Job: Fill out the Create Job form and click Create Job. The new job will appear in the list.
  3. Run a Job: Click Run Now next to any job to trigger it immediately. The job's status will update to running and then back to scheduled after a few seconds.
  4. Advanced Testing: Click Open Swagger UI (top right) to explore and test all API endpoints interactively, including viewing request/response schemas.

Create Job

How to create a job:
  • Name: Any label for your job (e.g., Backup).
  • Schedule (cron): Cron expression for when to run (e.g., @every 1m or 0 0 * * * for daily at midnight).
  • Command: Any text (e.g., echo hello). This is just a label; no real command is run.
Example:
{
  "name": "Example Job",
  "schedule": "@every 1m",
  "command": "echo hello"
}
